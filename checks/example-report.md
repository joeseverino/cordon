# Example: a checks run report

This is what cordon's checks engine produces over a repo — the human render of the
exact verdict an agent reads as [`--json`](README.md#the---json-contract). One
status table shows *what ran, what skipped and why, and what failed*; each failure
carries its `fix` and the command to `rerun` it. It's
[generated on every run](README.md#the-report-cordon-checks-reportmd) and surfaced
to the CI run summary, so a green badge and a red one both *explain themselves*.

## A failing run

A real run, rendered live in a GitHub Actions run summary — a passing unit suite,
a failing type check (output folded into the failure section), and a Playwright
spec that **skips and says why** (`requires playwright, built-dir`) rather than
silently not running:

## A passing run

Green is just as legible — the table still shows the skips and their reasons, so a
*skip* is never mistaken for a *pass*:

> **Cordon checks — all passed**
>
> 2 passed · 0 failed · 2 skipped
>
> | | check | effect | note |
> |:--:|---|---|---|
> | ✅ | Repository Policy | `read` | |
> | ✅ | Worktree Idempotence | `local_write` | |
> | ⏭️ | Internal Link Integrity | `read` | requires built-dir |
> | ⏭️ | Structural HTML | `read` | requires built-dir |
>

## How to read it

- **status** — ✅ pass · ❌ fail · ⏭️ skip. A skip is *fail-soft*: the check
  couldn't run here (a missing tool, no build output, wrong platform), not that it
  failed.
- **effect** — the check's blast radius on cordon's ladder
  (`read < local_write < vault_write < remote_write < deploy`) — the one signal an
  agent risk-gates on before acting.
- **note** — for a skip, the capability it `requires`; for a failure, the one-line
  `fix`. The full failure output is folded into the per-failure section, with the
  exact command to reproduce it.

The same run is also a machine verdict: `node "$CORDON_HOME/checks/run.mjs"
--root "$PWD" --json` emits [`cordon-checks-v2`](../schema/cordon-checks-v2.json),
the report and the JSON being two renders of one source. See
[`checks/README.md`](README.md) for the engine.
